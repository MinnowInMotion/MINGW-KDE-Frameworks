5 ## Introduction
6 
7 This framework provides plugins for Qt Designer that allow it to display
8 the widgets provided by various KDE frameworks, as well as a utility
9 (kgendesignerplugin) that can be used to generate other such plugins
10 from ini-style description files.
11 
12 ## kgendesignerplugin
13 
14 To use kgendesignerplugin in your own project, add
15 
16  find_package(KF5DesignerPlugin)
17 
18 to your CMakeLists.txt file, and use the
19 kf5designerplugin\_add\_widget\_files macro. For example, you might do
20 
21  find_package(Qt5Designer)
22  find_package(KF5DesignerPlugin NO_MODULE)
23  include_directories(${Qt5Designer_INCLUDE_DIRS})
24  if(Qt5Designer_FOUND AND KF5DesignerPlugin_FOUND)
25  set(foowidgets_SRCS)
26  kf5designerplugin_add_widget_files(foowidgets_SRCS
27  foo.widgets
28  )
29  qt5_add_resources(foowidgets_SRCS
30  foowidgets.qrc # preview images
31  )
32  add_library(foowidgets MODULE ${foowidgets_SRCS})
33  target_link_libraries(foowidgets
34  foo # library containing the actual widget classes
35  )
36  install(TARGETS foowidgets DESTINATION ${QT_PLUGIN_INSTALL_DIR}/designer)
37  endif()
38 
39 foo.widgets is an ini-style configuration file (see the KConfig
40 framework for format details) describing the widgets exported by the
41 plugin. It must include a [Global] section providing a PluginName entry
42 (the value of which must be a valid C++ class name). For example,
43 
44  [Global]
45  PluginName=FooWidgets
46 
47 This section can also list additional headers that should be included
48 for the plugin code to work, for example preview classes (see ImplClass
49 documentation below), with the Includes entry. It can also specify a
50 default value for the Group option (see below) with the DefaultGroup
51 entry.
52 
53  [Global]
54  PluginName=FooWidgets
55  Includes=classpreviews.h,otherinclude.h
56  DefaultGroup=Foo Widgets
57 
58 In addition, it should have a section for each widget you wish to
59 include in the plugin. For example, if you have a class FooView (that
60 inherits QWidget), whose header file is fooview.h, you could have the
61 section
62 
63  [FooView]
64  IncludeFile=fooviewwidget.h
65  ImplClass=FooViewPreview
66  Group=Views (Foo)
67 
68 The implementation of every method of QDesignerCustomWidgetInterface can
69 be controlled using options similar to the Group entry in the above
70 example, which causes the group() method to return "Views (Foo)".
71 
72 kgendesignerplugin attempts to pick sensible defaults for these values:
73 
74 - IncludeFile: would default to "fooview.h" if it were omitted from the
75  example
76 - Group: defaults to the value of DefaultGroup in the [Global] section,
77  which itself defaults to "Custom"
78 - ToolTip: defaults to "FooView Widget" in the example
79 - WhatsThis: defaults to "FooView Widget" in the example
80 - IsContainer: defaults to false
81 - CreateWidget: defaults to `return new ImplClass ConstructorArgs` where
82  ImplClass and ConstructorArgs are entries (see below)
83 - DomXml: defaults to the value of
84  QDesignerCustomWidgetInterface::domXml()
85 - Initialize: the default simply sets the mInitialized variable to true
86  if it is not already
87 
88 Note that no Icon entry is given above, instead you should provide an
89 icon name with IconName; this defaults to ":/pics/fooview.png" in the
90 above example. You should include this image in a resource file.
91 
92 If your class requires arguments other than the parent widget, or you
93 need to use a different class for previewing the widget in Qt Designer
94 (FooViewPreview in the above example), rather than setting CreateWidget
95 you can use some combination of ImplClass and ConstructorArgs:
96 
97 - ImplClass: would default to FooView if it were omitted from the
98  example
99 - ConstructorArgs: defaults to (parent) - note that the parentheses
100  are required
