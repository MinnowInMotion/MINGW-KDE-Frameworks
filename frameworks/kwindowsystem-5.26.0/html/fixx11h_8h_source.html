30 /* Usage:
31 
32  If you get compile errors caused by X11 includes (the line
33  where first error appears contains word like None, Unsorted,
34  Below, etc.), put #include <fixx11h.h> in the .cpp file
35  (not .h file!) between the place where X11 headers are
36  included and the place where the file with compile
37  error is included (or the place where the compile error
38  in the .cpp file occurs).
39 
40  This file remaps X11 #defines to const variables or
41  inline functions. The side effect may be that these
42  symbols may now refer to different variables
43  (e.g. if X11 #defined NoButton, after this file
44  is included NoButton would no longer be X11's
45  NoButton, but Qt::NoButton instead). At this time,
46  there's no conflict known that could cause problems.
47 
48  The original X11 symbols are still accessible
49  (e.g. for None) as X::None, XNone, and also still
50  None, unless name lookup finds different None
51  first (in the current class, etc.)
52 
53  Use 'Unsorted', 'Bool' and 'index' as templates.
54 
55 */
