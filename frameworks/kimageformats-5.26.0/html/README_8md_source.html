3 Plugins to allow QImage to support extra file formats.

5 ## Introduction
6 
7 This framework provides additional image format plugins for QtGui. As
8 such it is not required for the compilation of any other software, but
9 may be a runtime requirement for Qt-based software to support certain
10 image formats.
11 
12 ## Formats
13 
14 The following image formats have read-only support:
15 
16 - Gimp (xcf)
17 - OpenEXR (exr)
18 - Photoshop documents (psd)
19 - Sun Raster (ras)
20 
21 The following image formats have read and write support:
22 
23 - Encapsulated PostScript (eps)
24 - Personal Computer Exchange (pcx)
25 - SGI images (rgb, rgba, sgi, bw)
26 - Softimage PIC (pic)
27 - Targa (tga): supports more formats than Qt's version
28 - XView (xv)
29 
30 ## Contributing
31 
32 See the QImageIOPlugin documentation for information on how to write a
33 new plugin.
34 
35 The main difference between this framework and the qimageformats module
36 of Qt is the license. As such, if you write an imageformat plugin and
37 you are willing to sign the Qt Project contributor agreement, it may be
38 better to submit the plugin directly to the Qt Project.
39 
40 Note that the imageformat plugins provided by this module also provide a
41 desktop file. This is for the benefit of KImageIO in the KDE4 Support
42 framework.
43 
44 ## Duplicated Plugins
45 
46 The TGA plugin supports more formats than Qt's own TGA plugin;
47 specifically, the one provided here supports indexed, greyscale and RLE
48 images (types 1-3 and 9-11), while Qt's plugin only supports type 2
49 (RGB) files.
50 
51 The code for this cannot be contributed upstream directly because of
52 licensing. If anyone were willing to write fresh code to improve Qt's
53 TGA plugin, it would allow the TGA plugin in this framework to be
54 removed.
