3 Plugin framework for user interface components

5 ## Introduction
6 
7 This library implements the framework for KDE parts, which are
8 elaborate widgets with a user-interface defined in terms of actions
9 (menu items, toolbar icons).
10 
11 
12 ## Usage
13 
14 If you are using CMake, you need to have
15 
16  find_package(KF5Parts NO_MODULE)
17 
18 (or similar) in your CMakeLists.txt file, and you need to link to KF5::Parts.
19 
20 The main class is KParts::Part. This, or one of its subclasses, is what authors
21 of parts implement, and how users of parts interact with them.
22 
23 Applications wishing to embed a part need to inherit their main window from
24 KParts::MainWindow and provide a so-called shell GUI, which provides a basic
25 skeleton GUI with part-independent functionality/actions.
26 
27 Some KParts applications won't be specific to a given part, but expect
28 to be able to embed, for instance, all types of viewers out there. For this
29 the basic functionality of any viewer has been implemented in
30 KParts::ReadOnlyPart, which viewer-like parts should inherit from.
31 The same applies to KParts::ReadWritePart, which is for editor-like parts.
32 
33 You can add actions to an existing KParts app from "outside", defining
34 the code for those actions in a shared library. This mechanism is
35 obviously called plugins, and implemented by KParts::Plugin.
36 
37 For a complete, and very simple, example of how to use KParts to display
38 any kind of file (i.e. making a generic viewer), see the documentation for
39 KParts::ComponentFactory::createPartInstanceFromQuery.
