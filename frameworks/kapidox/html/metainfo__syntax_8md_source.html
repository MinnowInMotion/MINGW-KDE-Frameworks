1 # Description of the metainfo.yml syntax {#metainfo_syntax}
2 The name is the folder name.
3 The fancyname is the project name used in CMake.
4 
5 Each group must contain one and only one library with the `group_info` block. If
6 more are defined, the behavior is unknown.
7 
8 **Be sure to to set `public_lib` to true, or the library will be ignored.** Most of
9 the other keys are optionals. See below for a simplier example.
10 
11 ## All possible keys.
12 
13 ```yaml
14 description: Library doing X and Y
15 fancyname: The FancyName # optional, else replaced by the `project()` value of the `CMakeLists.txt`
16 maintainer: ochurlaud # optional, replaced by The KDE Community
17 group: frameworks # optional
18 subgroup: tier 1 # optional, a group must be defined
19 type: functional # optional
20 logo: relative/path/to/logo.png # Used only if not part of a group
21 platforms: # optional, name accepted: all, Linux, Windows, MacOSX
22  - name: Linux
23  note: Functional only with running BlueZ 5 # optional
24 public_lib: true # if not defined, the library is ignored
25 public_source_dirs: # optional, default to src, must be a list
26  - src1
27  - src2
28 public_doc_dir: docs # optional, default to docs
29 public_example_dir: examples # optional, default to examples
30 
31 portingAid: true/false # optional, default to false
32 deprecated: true/false # optional, default to false
33 libraries: KF5::MyLib # optional
34 
35 libraries: # optional
36  - qmake: BluezQt
37  cmake: KF5::BluezQt
38 cmakename: KF5BluezQt # optional
39 
40 irc: kde # optional, overwrite group field, if both not defined, default to kde-devel
41 mailinglist: mylib-dev # optional, overwrite group field, if both not defined, default to kde-devel
42 
43 group_info: # optional, only once per group
44  name: frameworks # optional, else the name is defined by the 'group' key above.
45  fancyname: The KDE Frameworks
46  maintainer: dfaure
47  irc: kde-devel
48  mailinglist: kde-core-devel
49  platforms:
50  - Linux
51  - Windows (partial)
52  - MacOSX (partial)
53  description: Providing everything from simple utility classes to ..
54  logo: relative/path/to/logo.png
55  long_description: # optional, one item per paragraph, html authorized
56  - The KDE Frameworks build on the <a href="http://qt-project.org">Qt
57  framework</a>, providing everything from simple utility classes (such as
58  those in KCoreAddons) to integrated solutions for common requirements of
59  desktop applications (such as KNewStuff, for fetching downloadable add-on
60  content in an application, or the powerful KIO multi-protocol file access
61  framework).
62  - ....
63  subgroups: # every subgroup must be declared here or will be ignored
64  - name: Tier 1
65  description: Tier 1 frameworks depend only on Qt (and possibly a small number of other third-party libraries), so can easily be used by an Qt-based project.
66  - name: Tier 2
67  description: Tier 2 frameworks additionally depend on tier 1 frameworks, but still have easily manageable dependencies.
68  - ..
69 ```
70 
71 ## Simple example 1
72 
73 In this case, the library is not part of a group, and the source is in `src`. If
74 there is no logo, a default one is used.
75 
76 ```yaml
77 description: Library doing X
78 maintainer: ochurlaud
79 public_lib: true
80 logo: libX.png
81 ```
82 
83 ## Simple example 2
84 
85 In this case, the libraries X and Y are part of the same group `XY`, and the
86 source is in `src`.
87 
88 ### libX
89 ```yaml
90 description: Library doing X
91 maintainer: ochurlaud
92 public_lib: true
93 platforms:
94  - name: Linux
95  - name: Windows
96  note: doesn't work with Windows10
97 group: XY
98 
99 group_info:
100  fancyname: XY Libraries
101  maintainer: arandomguy
102  platforms:
103  - Linux
104  - Windows (partial)
105  description: XY provides ...
106  logo: logo_XY.png
107  long_description:
108  - XY does this and this
109  - If you want to contribute please write to ...
110 ```
